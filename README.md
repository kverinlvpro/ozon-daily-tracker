# Ozon Daily Cover Tracker

Google-таблица на Apps Script: менеджеры Ozon видят, **как меняется обложка товара день ото дня**, и ежедневные метрики под ней — CTR, показы, конверсию в корзину, CR корзина→заказ, заказы, выкупы.

## Что умеет

- **Листы по кабинетам** — отдельный лист на каждый кабинет Ozon.
- **Блочная раскладка как в макете**: каждый артикул — блок строк. Верхняя строка — обложка (по картинке на каждый день, видно момент смены). Ниже строки метрик и ручные строки (Гипотеза / Вид продвижения / Комментарий). Каждый столбец = один день.
- **Обновление раз в день или по кнопке** (меню → ▶️ Обновить данные сейчас, либо ежедневный триггер).
- **Предупреждения** (лист «⚠️ Предупреждения» + подсветка прямо в листах):
  - после смены обложки конверсия **упала/выросла** больше порога;
  - обложка **не менялась дольше N дней**.

## Развёртывание (clasp)

Проект пока без `.clasp.json` — он создаётся под новую таблицу:

```bash
cd C:/projects/ozon-daily-tracker
clasp login                      # один раз
clasp create --type sheets --title "Ozon Daily Cover Tracker"
clasp push
```

`clasp create --type sheets` сам создаст новую Google-таблицу и привязанный скрипт.

Затем в таблице:

1. Меню **📸 Трекер обложек Ozon → 🆕 Инициализировать таблицу**.
2. Заполни лист **Аккаунты** (Client-Id, Api-Key, при наличии — `client_id|secret` для Performance). Поле **Название** становится именем листа кабинета.
3. Заполни лист **Артикулы** (SKU/offer_id, ID аккаунта, менеджер, статус «активен»).
4. **🔧 Тест подключения к Ozon API** — проверь, что метрики тянутся.
5. **▶️ Обновить данные сейчас** — появятся листы кабинетов.
6. **⏰ Установить ежедневный триггер**.

## Настройки (лист «Настройки»)

| Ключ | По умолчанию | Смысл |
|---|---|---|
| `lag_days` | 1 | За какой день брать метрики (аналитика Ozon с задержкой ~1 день) |
| `update_hour` | 7 | Час ежедневного автообновления |
| `display_days` | 60 | Сколько последних дней показывать колонками |
| `stale_cover_days` | 14 | Порог «обложка залежалась» |
| `alert_threshold_pct` | 10 | Порог изменения конверсии для алерта, % |
| `before_window_days` | 7 | Окно ДО смены обложки для сравнения |
| `after_window_days` | 7 | Окно ПОСЛЕ смены обложки для сравнения |

## Примечания

- Обложка снимается на момент запуска, метрики — за `lag_days` назад. При ежедневном запуске колонки соответствуют календарным дням.
- Источник истины — скрытый лист `_Данные`. Листы кабинетов перерисовываются из него; ручные строки сохраняются по примечаниям-датам в заголовке.
- CTR рекламы требует заполненных Performance-кредов; без них CTR = 0, остальные метрики работают.
